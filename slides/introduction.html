sphinxcontrib-mixed-builder

なにこれ？

変わり種のSphinx拡張の1つで、

「複数のビルダーを組み合わせ使う」

ビルダーを提供します。

どういうこと？

- index.rst
- usage.rst
- slides/
  - introduce.rst
  - sample-1.rst
  - sample-2.rst

このような構成のSphinxプロジェクト

どういうこと？

- index.rst       -> html
- usage.rst       -> html
- slides/
  - introduce.rst -> html
  - sample-1.rst  -> html
  - sample-2.rst  -> html

make html だと

どういうこと？

- index.rst       -> revealjs
- usage.rst       -> revealjs
- slides/
  - introduce.rst -> revealjs
  - sample-1.rst  -> revealjs
  - sample-2.rst  -> revealjs

make revealjs だと

どういうこと？

- index.rst       -> html
- usage.rst       -> html
- slides/
  - introduce.rst -> revealjs
  - sample-1.rst  -> revealjs
  - sample-2.rst  -> revealjs

make mixed (+適切な設定)なら

どういうこと？

単一ビルドなのに、HTMLとReveal.jsを同居させることが出来る！

使い方

PyPI未登録です

pip install git+https://github.com/attakei-lab/sphinx-mixed-builder

conf.py

本拡張+他のビルダー拡張を追加する

extensions = [
    "sphinxcontrib.mixed_builder",
    "sphinx_revealjs",
]

conf.py

使うビルダー群を指定する

mixed_builders = ["html", "revealjs"]

conf.py

ビルダーの適用ルールを指定する

mixed_rules = [
    {
        "docname": "slides/introduction",
        "builder": "revealjs",
    }
]

ビルドする

make コマンドでビルドしつつ、Pythonのサーバーモジュールで表示確認してみましょう

make mixed
python -m http.server -d _build/mixed

おまけ

なんでこんなの作ったの？

sphinx-revealjs で複数のスライドを管理する際に、「インデックスもReveal.js化する」問題の解消をしたかった。
